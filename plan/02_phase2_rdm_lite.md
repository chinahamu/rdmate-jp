# 02. フェーズ2：RDM Lite（研究データ台帳）

> 目標: DMPで記載した方針を、研究期間中に発生するデータセット・保存場所・公開可否・メタデータ管理へ接続し、研究室単位で使える軽量RDM機能を完成させる。

---

## 0. Phase 2 ゴール定義（Definition of Done）

- 研究課題ごとにデータセット台帳を作成できる。
- データセット単位で保存場所、責任者、公開区分、ライセンス、保持期間、関連論文、関連コードを管理できる。
- DMPで宣言した方針とデータセット実態の差分を検出できる。
- リポジトリ登録に必要な基本メタデータをJSON/CSVで出力できる。
- 実データファイルそのものは保持せず、パス・URL・外部IDの参照管理を基本とする。

## 4. 公開区分・ライセンス管理

### 4.1 公開区分

- [x] `open`: 公開可能
- [x] `embargoed`: 一定期間後に公開
- [x] `restricted`: 申請制・制限付き公開
- [x] `closed`: 非公開
- [x] `undecided`: 未定

### 4.2 ライセンス候補

- [x] CC BY 4.0
- [x] CC BY-SA 4.0
- [x] CC0 1.0
- [x] MIT
- [x] Apache-2.0
- [x] GPL-3.0
- [x] Custom / Not specified

### 4.3 公開判断チェック

- [x] 個人情報を含む可能性がある場合、公開区分 `open` に警告する。
- [x] 共同研究契約ありの場合、公開予定日・利用条件の入力を促す。
- [x] 特許出願予定ありの場合、公開タイミング確認を促す。
- [x] `closed` の場合、非公開理由を必須化する。

## 5. DMPとの差分チェック

- [x] DMPで定めた保存場所が各データセットに設定されているか。
- [x] DMPで定めた公開方針とデータセットの公開区分が矛盾していないか。
