# 01. フェーズ1：MVP（DMP Builder）

> 目標: 研究課題を登録し、助成金申請・研究開始時に必要な DMP の初版を作成、検証、エクスポートできる最小構成を完成させる。

---

## 0. Phase 1 ゴール定義（Definition of Done）

- 研究課題・助成金・研究メンバーを登録できる。
- DMPテンプレートを選択し、フォーム入力できる。
- 入力内容を Markdown / JSON / CSV / DOCX で出力できる。
- 必須項目、公開方針、保存場所、責任者、保持期間の未入力チェックができる。
- SQLite に全データを永続化し、ローカル環境で動作する。
- Playwright による主要E2E（プロジェクト作成→DMP出力）が通る。

---

## 1. プロジェクト初期化

### 1.1 リポジトリ構成

- [x] Next.js + TypeScript プロジェクトを初期化する。
- [x] `src/` 配下に app / components / lib / server / domain を分離する。
- [x] `prisma/` または `drizzle/` にDBスキーマとマイグレーションを配置する。
- [x] `templates/dmp/` にDMPテンプレート定義を配置する。
- [x] `docs/` に利用者向けドキュメントを配置する。

推奨構成:

```text
rdmate-jp/
  src/

    components/
    domain/
    server/
    lib/
  templates/
    dmp/
  prisma/ or drizzle/
  tests/
    unit/
    e2e/

  docs/
```

### 1.2 開発環境

- [x] Node.js LTS を前提にする。
- [x] `pnpm` を標準パッケージマネージャにする。
- [x] ESLint / Prettier / TypeScript strict を有効化する。
- [x] `.env.example` を追加する。
- [x] GitHub Actions で `lint`, `typecheck`, `test`, `build` を実行する。

## 3. DMPテンプレート基盤

### 3.1 テンプレート定義フォーマット

- [x] DMPテンプレートをJSON/YAMLで定義する。
- [x] セクション、質問、入力型、必須/任意、ヘルプテキスト、出力順を持つ。
- [x] Zodでテンプレート定義を検証する。
- [x] テンプレートIDとバージョンを管理する。

例:

```json
{
  "id": "generic-jp-v1",
  "name": "汎用DMP 日本語テンプレート",
  "version": "1.0.0",
  "sections": [
    {
      "id": "data_description",
      "title": "研究データの概要",
      "questions": [
        {
          "id": "data_types",
          "label": "生成・収集するデータの種類",
          "type": "textarea",
          "required": true
        }
      ]
    }
  ]
}
```

### 3.2 MVPテンプレート

- [x] `generic-jp-v1`: 日本語汎用テンプレート。
- [x] `university-basic-v1`: 大学研究室向け簡易テンプレート。
- [x] `nih-style-v1`: NIH DMS Plan 風の英日併記テンプレート。

### 3.3 テンプレート管理UI

- [x] 利用可能テンプレート一覧を表示する。
- [x] テンプレート詳細、対象助成機関、項目数、最終更新日を表示する。
- [x] プロジェクト作成時にテンプレートを選択できる。

### 4.2 入力補助

### 4.3 バリデーション

- [x] URL形式、日付範囲、メール形式を検証する。

## 5. DMPエクスポート

### 5.1 Markdown出力

- [x] セクション順にMarkdownを生成する。

- [x] フロントからプレビュー表示できる。

- [x] 1行1設問形式で出力する。
- [x] `section_id`, `question_id`, `question_label`, `answer`, `required`, `status` を含める。
- [x] URA・図書館担当者が表計算で確認しやすい形式にする。

### 5.4 DOCX出力

- [x] MarkdownからDOCXへ変換する。
- [x] 表紙情報（研究課題名、PI、助成機関、生成日）を入れる。
