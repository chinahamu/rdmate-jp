# 依存ライブラリ更新手順

RDMate JPの依存ライブラリ更新は、運用中の研究課題・DMP・Dataset台帳に影響しないよう、検証ブランチで実施します。

## 更新フロー

1. `main` から更新用ブランチを作成する。
2. `pnpm outdated` で更新候補を確認する。
3. 小さな単位で `pnpm update <package>` を実行する。
4. `pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm build` を実行する。
5. Prisma、Next.js、React、Zodなど、実行時影響が大きい依存関係はCHANGELOGを確認する。
6. 破壊的変更がある場合は `docs/migration.md` に移行手順を追記する。
7. PR本文に更新理由、検証結果、ロールバック方針を記載する。

## セキュリティ更新

セキュリティ修正は通常更新より優先します。ただし、学内サーバ運用では事前にステージング環境で起動確認し、バックアップ取得後に本番適用してください。

## ロールバック

更新後に問題が出た場合は、直前のlockfileとpackage.jsonに戻し、DBスキーマ変更がある場合はバックアップから復元できる状態を確認してください。
