# 権限・監査ログ・推奨設定

RDMate JPは、個人利用、研究室共有、学内サーバ運用を想定し、利用モードとロールに応じて研究課題単位の操作権限を管理します。このドキュメントでは、権限設計、監査ログ、推奨設定を説明します。

## 1. 利用モード

| モード | 想定 | 特徴 |
| --- | --- | --- |
| `local` | 個人PC・研究室PC | 単一ユーザー、ローカルSQLite、認証なしまたは簡易認証 |
| `team` | 研究室内共有 | 複数ユーザー、研究課題単位の権限、監査ログ確認 |
| `institution` | 学内サーバ | 管理者、PI、Data Steward、複数研究課題、HTTPS前提 |

`RDMATE_USAGE_MODE` で運用モードを設定します。共有・学内運用では、HTTPS、バックアップ、セッション設定、監査ログ保全を必須にしてください。

## 2. ロール

| ロール | 役割 |
| --- | --- |
| Admin | システム設定、権限、利用モード、全体監査を管理します。 |
| PI | 研究課題の責任者としてDMP、Dataset、公開判断を承認します。 |
| Data Steward | DMP、メタデータ、公開判断、リポジトリ登録を支援します。 |
| Member | 研究課題に参加し、Datasetやメタデータを入力します。 |
| Viewer | 閲覧・確認を中心に行い、編集権限は持ちません。 |

## 3. 操作権限

研究課題ごとに次の操作権限を判定します。

| 権限 | 内容 |
| --- | --- |
| `view_project` | 研究課題を閲覧できる |
| `edit_dmp` | DMPを作成・更新できる |
| `edit_dataset` | Dataset台帳、保存場所、メタデータを編集できる |
| `export_project` | CSV/JSON/Markdownなどを出力できる |
| `manage_settings` | 研究課題の設定、権限、管理項目を変更できる |

UIだけでなくAPI直接呼び出しでも、サーバ側で権限チェックを強制します。

## 4. 推奨ロール割当

### local

- AdminまたはPI相当の単一ユーザーを想定します。
- 実験・検証用途ではサンプルデータで開始します。
- 外部公開URLには配置しないでください。

### team

- PIを研究課題責任者に設定します。
- Data Stewardを1名以上設定します。
- 学生・共同研究者はMemberまたはViewerにします。
- エクスポート権限はPIとData Stewardを基本にします。

### institution

- Adminは最小人数にします。
- 研究課題ごとにPIとData Stewardを設定します。
- Viewerは閲覧範囲を研究課題単位で限定します。
- 監査ログの定期確認とバックアップを運用ルールに含めます。

## 5. 監査ログ

監査ログでは、誰が、いつ、何を、どの研究課題に対して行ったかを記録します。

主な記録対象:

- 研究課題作成・更新・削除
- DMP作成・更新・エクスポート
- Dataset作成・更新・削除
- 公開区分・ライセンス変更
- 外部連携設定変更
- 権限設定変更
- エクスポート履歴

## 6. 変更履歴

DMP回答とDatasetメタデータは、変更前/変更後の差分を保持します。

確認項目:

- 変更者
- 変更日時
- 対象エンティティ
- 変更前の値
- 変更後の値
- バージョンタグ
- 変更理由

研究終了時や公開前レビューでは、DMPとDatasetの変更履歴を確認してください。

## 7. エクスポート履歴

エクスポート履歴では、次を記録します。

- 出力者
- 出力日時
- 形式
- 研究課題
- ファイル名
- ファイルハッシュ
- 前回との差分

機関リポジトリ登録前、研究終了時、監査対応時に利用します。

## 8. 推奨設定チェックリスト

### local

- [ ] `.env` が作成されている。
- [ ] SQLite DBの保存場所を把握している。
- [ ] バックアップ先を決めている。
- [ ] 実データ本体を保存しない運用を理解している。

### team

- [ ] PI、Data Steward、Member、Viewerを割り当てている。
- [ ] エクスポート権限を必要最小限にしている。
- [ ] 監査ログの確認担当者を決めている。
- [ ] バックアップ取得頻度を決めている。

### institution

- [ ] HTTPSでのみ利用できる。
- [ ] `SESSION_SECRET` をランダム値に変更している。
- [ ] Admin権限者を最小化している。
- [ ] バックアップと復元リハーサルを実施している。
- [ ] 監査ログの保存期間を決めている。
- [ ] 学内規程・研究倫理・契約上の公開制約を確認している。

## 9. API確認

権限サマリーは次のAPIで確認できます。

```bash
curl http://localhost:3000/api/access-control/summary
```

監査ログ・変更履歴・エクスポート履歴のサマリーは次で確認できます。

```bash
curl http://localhost:3000/api/audit-history/summary
```
