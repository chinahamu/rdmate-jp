# RDM Liteの使い方

RDM Liteは、研究データ本体ではなく、研究データの管理情報を台帳化するための機能です。Dataset、保存場所、ファイルマニフェスト、メタデータ、公開区分、関連成果物を一連の流れで管理します。

## 1. 管理対象

RDMate JPが保持するのは次の管理情報です。

- 研究課題情報
- 研究メンバーと助成情報
- Dataset台帳
- 保存場所の参照情報
- ファイルマニフェスト
- メタデータ
- 公開区分、ライセンス、利用条件
- 関連論文、ソフトウェア、手順書
- DMPとの差分、エクスポート履歴、監査ログ

研究データ本体は、研究室サーバ、大学提供ストレージ、GakuNin RDM、機関リポジトリ、GitHubなど、適切な外部保管場所で管理してください。

## 2. Dataset台帳

`/datasets` では、研究課題ごとのデータセットを一覧・確認できます。

| 項目 | 説明 |
| --- | --- |
| データセット名 | 人が読んで識別できる名称 |
| 説明 | 収集・生成方法、対象、用途 |
| データ種別 | 調査データ、実験データ、画像、コードなど |
| 研究課題 | 紐づくResearchProject |
| 作成者 | データセットの作成者 |
| 責任者 | 管理責任者またはData Steward |
| 生成日・更新日 | 作成・更新の時点 |
| バージョン | データセットの管理バージョン |
| ステータス | 未確認、確認済み、公開準備中など |
| 公開区分 | open / embargoed / restricted / closed / undecided |

一覧では、データ種別、ステータス、公開区分、責任者、ライセンスで絞り込みます。

## 3. 保存場所・参照管理

`/storage` では、データセットの保存場所とファイルマニフェストを管理します。

保存場所の例:

- ローカルパス
- ネットワークドライブ
- クラウドストレージURL
- GakuNin RDM URL
- GitHub URL
- 機関リポジトリURL

保存場所には、アクセス範囲、バックアップ有無、暗号化有無、補足メモを付けます。URL形式、保存場所未設定、バックアップ未設定などは参照チェックで確認します。

## 4. ファイルマニフェスト

ファイルマニフェストは、実データ本体の代わりに、ファイルの所在と同定情報を管理するための一覧です。

| 項目 | 説明 |
| --- | --- |
| ファイル名 | ファイルの表示名 |
| 相対パス | 保存場所内の相対パス |
| サイズ | バイト単位のサイズ |
| ハッシュ | SHA-256などのハッシュ値 |
| MIMEタイプ | `text/csv`、`application/json` など |
| 更新日時 | ファイル最終更新日時 |

マニフェストはCSVまたはJSONで出力できます。

## 5. メタデータ管理

`/metadata` では、リポジトリ登録や成果物公開に使う基本メタデータを整理します。

- タイトル
- 説明
- 作成者
- 所属機関
- キーワード
- 研究分野
- 取得方法
- 時間範囲
- 空間範囲
- 変数・カラム説明
- 関連論文
- 関連ソフトウェア

研究課題情報、メンバー情報、助成金情報、Dataset情報から初期メタデータとキーワード候補を生成できます。

## 6. 公開区分・ライセンス

`/publication-license` で公開判断を確認します。

| 公開区分 | 用途 |
| --- | --- |
| `open` | 公開可能 |
| `embargoed` | 一定期間後に公開予定 |
| `restricted` | 条件付きで共有 |
| `closed` | 非公開 |
| `undecided` | 判断未定 |

ライセンス候補には、CC BY 4.0、CC BY-SA 4.0、CC0 1.0、MIT、Apache-2.0、GPL-3.0、Custom、Not specifiedがあります。

公開判断では、個人情報、共同研究契約、特許出願予定、公開予定日、利用条件、非公開理由を確認します。`closed` の場合は非公開理由を必ず記録します。

## 7. DMPとの差分チェック

`/dmp-diff` では、DMP方針とDataset台帳の差分を検出します。

主なチェック項目:

- 予定データ種別
- 保存場所
- 公開方針
- 保持期間・廃棄方針
- 責任者

差分はエラー、警告、情報に分類されます。DMPを更新するべき差分とDatasetを修正するべき差分を分けて確認してください。

## 8. 関連成果物

`/related-outputs` では、Datasetと次の成果物を紐付けます。

- 論文・発表: DOI、arXiv、雑誌名、出版年、Datasetとの関係
- ソフトウェア・コード: GitHub/GitLab URL、リリースタグ、ライセンス、実行環境
- 実験・調査手順: 手順書URL、プロトコルID、バージョン、責任者、変更履歴

## 9. 運用上の推奨

- Dataset登録時点で責任者を必ず設定します。
- 保存場所にはバックアップ有無を記録します。
- 公開判断が未定のデータは `undecided` のままにせず、期限と確認者を決めます。
- 研究終了時には完全JSON、リポジトリCSV、マニフェストをまとめて出力します。
