# RDMate JP Code of Conduct

## 1. 目的

RDMate JPは、研究データ管理を支援するOSSとして、研究者、大学・研究機関の管理者、Data Steward、開発者、ドキュメント作成者、DMPテンプレート作成者が安心して参加できる場を目指します。

この行動規範は、Issue、Pull Request、Discussions、レビュー、ドキュメント、イベント、その他RDMate JPに関連するコミュニケーションに適用します。

## 2. 期待される行動

参加者には、次の行動を期待します。

- 相手の専門分野、経験、所属、言語、文化的背景を尊重する。
- 技術的な指摘は、人格ではなく変更内容や根拠に向ける。
- 初心者、利用者、非開発者にも分かりやすい説明を心がける。
- 研究データ、個人情報、秘密情報を不用意に共有しない。
- 不確かな情報は断定せず、根拠や前提を明示する。
- 建設的なフィードバックを行い、改善案を歓迎する。
- 議論が複雑になった場合は、Issue、Discussion、PRに分けて整理する。

## 3. 認められない行動

次の行動は認められません。

- ハラスメント、差別、脅迫、侮辱、個人攻撃。
- 性的、暴力的、差別的、威圧的な表現。
- 個人情報、秘密情報、研究データ本体、認証情報の無断投稿。
- 意図的な荒らし、スパム、議論の妨害。
- 他者の成果や貢献を不当に軽視する行為。
- セキュリティ上の問題を公開Issueで詳細に悪用可能な形で投稿する行為。

## 4. 研究データ・個人情報への配慮

RDMate JPは研究データ管理を扱うため、通常のOSS以上に情報管理への配慮が必要です。

Issue、PR、Discussionsでは、次を投稿しないでください。

- 実データ本体。
- 個人情報、要配慮情報、患者情報、学生情報、共同研究者の非公開情報。
- APIキー、トークン、パスワード、`.env` の内容。
- 未公開論文、未公開研究計画、契約上非公開の資料。

再現例が必要な場合は、匿名化・合成データ・最小サンプルを使ってください。

## 5. セキュリティ報告

セキュリティ上の問題を見つけた場合は、公開Issueで悪用手順を詳細に投稿しないでください。まずメンテナーに非公開の連絡手段で共有し、影響範囲、再現条件、緩和策を整理します。

公開Issueで扱う場合は、悪用可能な詳細を避け、修正方針と影響範囲を中心に記載します。

## 6. 違反への対応

メンテナーは、行動規範に反する投稿や行為に対して、次の対応を行うことがあります。

1. 注意喚起または編集依頼。
2. 投稿の編集・非表示・削除依頼。
3. Issue、PR、Discussionのロック。
4. 一時的または恒久的な参加制限。
5. 必要に応じたGitHubへの通報。

対応は、影響範囲、再発性、悪質性、関係者の安全を考慮して判断します。

## 7. メンテナーの責任

メンテナーは、行動規範を公平に適用し、報告者や被害を受けた参加者の安全とプライバシーを尊重します。また、議論の透明性を保ちつつ、個人情報やセキュリティ情報の保護を優先します。

## 8. 適用範囲

この行動規範は、RDMate JPのGitHubリポジトリ、関連するIssue、Pull Request、Discussions、ドキュメント、コミュニティ運用に適用します。
